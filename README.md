# Voice Notes Management System

A high-fidelity Voice Notes Management System featuring user authentication, speech-to-text transcribing, and real-time translation. Users can record voice notes in **English, Hindi, or Telugu** directly from their browser, convert it into text, edit it manually, and translate it across these languages on demand.

## Technology Stack

### Backend
- **Java 21**
- **Spring Boot 4.1.x** (with Spring Security 7.0 & Jakarta validation)
- **Spring MVC & Hibernate (JPA)**
- **JWT (Json Web Tokens)** for stateless authentication
- **MySQL** for relational storage
- **Swagger/OpenAPI** for API documentation
- **Lombok & SLF4J**

### Frontend
- **React.js** (Vite template)
- **React Router Dom**
- **Axios**
- **Bootstrap 5 & Custom CSS**
- **Material Icons Round**
- **React Toastify** for status alerts

## Getting Started

### Prerequisites
- Java 21 JDK installed
- Node.js (version 18 or above) & npm installed
- MySQL Server running locally

### Database Setup
1. Open your MySQL client (e.g. MySQL Workbench) or terminal.
2. Verify that your MySQL server is running at `localhost:3306`.
3. Create a schema named `notes_app`:
   ```sql
   CREATE DATABASE IF NOT EXISTS notes_app;
   ```
4. *Optional:* If you want to seed data manually instead of letting the application seed it on start, you can run the queries defined in the [seed_data.sql](file:///c:/Users/Crafter/IdeaProjects/nwvs/seed_data.sql) script.

---

### Backend Setup & Running

1. **Verify Configurations:** Open [application.properties](file:///c:/Users/Crafter/IdeaProjects/nwvs/src/main/resources/application.properties) and ensure that your database credentials (`spring.datasource.username` and `spring.datasource.password`) are correct. (Default values are set to `root` / `root`).
2. **Build and Run:** From the root workspace directory, run:
   ```bash
   ./mvnw spring-boot:run
   ```
3. The server will start on port `8080`.
4. **Programmatic Seeding:** The application is configured to automatically seed two default users (`john@example.com` and `jane@example.com`, both with password `password123`) and several sample notes in the database upon the first startup.

---

### Frontend Setup & Running

1. Open a new terminal and navigate to the `/frontend` directory:
   ```bash
   cd frontend
   ```
2. Install npm dependencies:
   ```bash
   npm install
   ```
3. Start the Vite React development server:
   ```bash
   npm run dev
   ```
4. The frontend will start running at `http://localhost:5173`.
5. Open your browser and navigate to `http://localhost:5173` to explore the system!

## API Documentation & Testing

### Swagger UI
Swagger documentation is automatically generated by `springdoc-openapi` and is available once the backend is running.
- **Swagger URL:** [http://localhost:8080/swagger-ui.html](http://localhost:8080/swagger-ui.html)
- You can test endpoints directly in Swagger. To access protected endpoints, log in via `/api/auth/login`, copy the `token` string, and paste it into the **Authorize** dialog as a `bearer` token.

### Postman Collection
A complete Postman collection containing all REST requests with sample payloads, query filters, and JWT automatic header variables has been generated.
- **Postman file:** [voice_notes_system.postman_collection.json](file:///c:/Users/Crafter/IdeaProjects/nwvs/voice_notes_system.postman_collection.json)
- To use, import this file into Postman, trigger the **Login User** request, and the script will automatically set the `token` parameter for all other requests.

## Key Features Built-in

- **High-Fidelity UI/UX:** Stunning dark and light themes, smooth transitions, card layouts with category tags, pulsing mic record status, and gradient aesthetics.
- **Robust Security:** JWT-based stateless sessions, BCrypt passwords, role scopes, and CORS configurations.
- **Voice Transcription:** Transcribe voice messages using the browser's native Web Speech API in English, Hindi, and Telugu.
- **Translation Engine:** Instant translation between English, Telugu, and Hindi via MyMemory integration.
- **Dashboard Stats:** Notes counters, category progress statistics, and quick navigation.
